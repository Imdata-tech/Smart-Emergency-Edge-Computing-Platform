### 软件要求

| 软件 | 版本 | 说明 |
|------|------|------|
| JDK | 17+ | Java运行环境 |
| Maven | 3.6+ | 项目构建工具 |

## 本地开发环境部署

### 步骤1: 克隆项目

```bash
git clone https://github.com/your-username/cardio-guard-360.git
cd cardio-guard-360
```

### 步骤2: 初始化数据库

#### 2.1 安装MySQL

**macOS:**
```bash
brew install mysql@8.0
brew services start mysql@8.0
```

**Ubuntu/Debian:**
```bash
sudo apt update
sudo apt install mysql-server-8.0
sudo systemctl start mysql
```

**CentOS/RHEL:**
```bash
sudo yum install mysql-server
sudo systemctl start mysqld
```

#### 2.2 执行初始化脚本

```bash
mysql -u root -p < database/init.sql
```

或者手动执行:
```sql
CREATE DATABASE cardioguard DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE cardioguard;
-- 然后执行 init.sql 中的建表语句
```

### 步骤3: 安装和配置Redis

**macOS:**
```bash
brew install redis
brew services start redis
```

**Ubuntu/Debian:**
```bash
sudo apt install redis-server
sudo systemctl start redis
```

**Windows:**
下载并安装: https://github.com/microsoftarchive/redis/releases

### 步骤4: 安装和配置InfluxDB

**macOS:**
```bash
brew install influxdb
brew services start influxdb
```

**Ubuntu/Debian:**
```bash
wget -q https://repos.influxdata.com/influxdata-archive_compat.key
echo '393e8779c89ac8d958f81f942f9ad7fb82a25e133faddaf92e15b16e6ac9ce4c influxdata-archive_compat.key' | sha256sum -c && cat influxdata-archive_compat.key | gpg --dearmor | sudo tee /etc/apt/trusted.gpg.d/influxdata-archive_compat.gpg > /dev/null
echo 'deb [signed-by=/etc/apt/trusted.gpg.d/influxdata-archive_compat.gpg] https://repos.influxdata.com/debian stable main' | sudo tee /etc/apt/sources.list.d/influxdata.list
sudo apt update
sudo apt install influxdb2
sudo systemctl start influxdb
```

**初始化InfluxDB:**
访问 http://localhost:8086 进行初始化设置,记录Token。

### 步骤5: 配置应用

编辑 `cardio-guard-backend/src/main/resources/application.yml`:

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/cardioguard?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: your_mysql_password  # 修改为你的MySQL密码
    
  data:
    redis:
      host: localhost
      port: 6379
      password:  # 如果Redis设置了密码,请填写

influxdb:
  url: http://localhost:8086
  token: your_influxdb_token  # 修改为你的InfluxDB Token
  org: cardioguard
  bucket: health-data
```

### 步骤6: 编译和运行

```bash
cd cardio-guard-backend

# 编译项目
mvn clean package -DskipTests

# 运行应用
java -jar target/cardioguard-backend-1.0.0.jar
```

### 步骤7: 验证部署

访问 Swagger UI: http://localhost:8080/api/swagger-ui.html

如果能看到API文档界面,说明部署成功!

## 生产环境部署

### 1. 服务器准备

推荐使用Linux服务器(Ubuntu 20.04/22.04 LTS或CentOS 8)

### 2. 安装依赖软件

```bash
# 安装JDK 17
sudo apt update
sudo apt install openjdk-17-jdk

# 安装Maven
sudo apt install maven

# 安装MySQL 8.0
sudo apt install mysql-server-8.0

# 安装Redis
sudo apt install redis-server

# 安装InfluxDB 2.x
# (参考上面的InfluxDB安装步骤)
```

### 3. 配置防火墙

```bash
# 开放必要端口
sudo ufw allow 8080/tcp  # 应用端口
sudo ufw allow 3306/tcp  # MySQL(仅内网)
sudo ufw allow 6379/tcp  # Redis(仅内网)
sudo ufw allow 8086/tcp  # InfluxDB(仅内网)
sudo ufw enable
```

### 4. 数据库配置优化

编辑 `/etc/mysql/mysql.conf.d/mysqld.cnf`:

```ini
[mysqld]
max_connections = 500
innodb_buffer_pool_size = 2G
query_cache_size = 64M
```

### 5. 应用配置优化

创建 `application-prod.yml`:

```yaml
server:
  port: 8080
  
spring:
  datasource:
    hikari:
      maximum-pool-size: 20
      minimum-idle: 5
      connection-timeout: 30000
      
logging:
  level:
    com.cardioguard: INFO
  file:
    name: /var/log/cardioguard/app.log
```

### 6. 打包上传

```bash
# 本地打包
mvn clean package -DskipTests

# 上传到服务器
scp target/cardioguard-backend-1.0.0.jar user@server:/opt/cardioguard/
```

### 7. 使用Systemd管理服务

创建 `/etc/systemd/system/cardioguard.service`:

```ini
[Unit]
Description=CardioGuard 360 Backend Service
After=syslog.target network.target mysql.service redis.service influxdb.service

[Service]
Type=simple
User=cardioguard
Group=cardioguard
WorkingDirectory=/opt/cardioguard
ExecStart=/usr/bin/java -jar /opt/cardioguard/cardioguard-backend-1.0.0.jar --spring.profiles.active=prod
ExecStop=/bin/kill -15 $MAINPID
Restart=always
RestartSec=10

StandardOutput=append:/var/log/cardioguard/app.log
StandardError=append:/var/log/cardioguard/error.log

[Install]
WantedBy=multi-user.target
```

启动服务:
```bash
sudo systemctl daemon-reload
sudo systemctl enable cardioguard
sudo systemctl start cardioguard
sudo systemctl status cardioguard
```

### 8. 配置Nginx反向代理(可选)

```nginx
server {
    listen 80;
    server_name api.cardioguard.com;

    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # WebSocket支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

## Docker一键部署

### 前置条件

- 安装Docker: https://docs.docker.com/get-docker/
- 安装Docker Compose: https://docs.docker.com/compose/install/

### 部署步骤

```bash
# 1. 克隆项目
git clone https://github.com/your-username/cardio-guard-360.git
cd cardio-guard-360

# 2. 启动所有服务
docker-compose up -d

# 3. 查看运行状态
docker-compose ps

# 4. 查看日志
docker-compose logs -f backend
```

### 常用命令

```bash
# 停止所有服务
docker-compose down

# 重启某个服务
docker-compose restart backend

# 进入容器
docker exec -it cardioguard-backend sh

# 查看数据库日志
docker logs cardioguard-mysql

# 清理资源(慎用)
docker-compose down -v
```

## 常见问题

### 1. 数据库连接失败

**问题:** `Communications link failure`

**解决方案:**
- 检查MySQL是否启动: `systemctl status mysql`
- 检查用户名密码是否正确
- 检查防火墙设置
- 确认MySQL允许远程连接

### 2. Redis连接失败

**问题:** `Cannot get Jedis connection`

**解决方案:**
- 检查Redis是否启动: `systemctl status redis`
- 检查Redis配置文件中的bind设置
- 确认Redis密码(如果有)

### 3. InfluxDB认证失败

**问题:** `Authorization failed`

**解决方案:**
- 检查Token是否正确
- 确认Organization和Bucket名称
- 重新生成Admin Token

### 4. 端口被占用

**问题:** `Port 8080 was already in use`

**解决方案:**
```bash
# 查找占用端口的进程
lsof -i :8080

# 杀死进程或更换端口
kill -9 <PID>
```

### 5. 内存不足

**问题:** `OutOfMemoryError`

**解决方案:**
```bash
# 调整JVM参数
java -Xms512m -Xmx2g -jar app.jar
```

### 6. WebSocket连接失败

**问题:** WebSocket无法建立连接

**解决方案:**
- 检查Nginx配置是否支持WebSocket
- 确认防火墙允许WebSocket通信
- 检查CORS配置

## 性能优化建议

### 1. 数据库优化
- 添加合适的索引
- 定期清理历史数据
- 使用读写分离

### 2. 缓存优化
- 合理使用Redis缓存
- 设置合适的过期时间
- 使用本地缓存减少网络开销

### 3. 应用优化
- 启用GZIP压缩
- 使用连接池
- 异步处理耗时操作

### 4. 监控告警
- 部署Prometheus + Grafana
- 设置关键指标告警
- 日志集中管理(ELK)

## 安全建议

1. **修改默认密码**: MySQL、Redis、InfluxDB都要修改默认密码
2. **启用HTTPS**: 生产环境必须使用HTTPS
3. **限制访问IP**: 数据库和缓存只允许内网访问
4. **定期更新**: 及时更新依赖包和安全补丁
5. **日志审计**: 开启操作日志和访问日志
6. **防火墙**: 只开放必要的端口

## 数据库选型

### MySQL (业务数据库)
- **版本**: 8.0+
- **用途**: 存储用户、设备、预警等业务数据
- **特点**: ACID事务、关系型数据、强一致性

### InfluxDB (时序数据库)
- **版本**: 2.7+
- **用途**: 存储心率、ECG等时序监测数据
- **特点**: 高写入性能、时间序列查询、数据压缩

### Redis (缓存数据库)
- **版本**: 7.x
- **用途**: 会话缓存、实时数据缓冲
- **特点**: 高性能、键值存储

## 表结构设计

### 1. sys_user (用户表)

**用途**: 存储系统用户信息,支持患者、医生、管理员三种角色

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | BIGINT | PK, AUTO_INCREMENT | - | 用户ID |
| username | VARCHAR(50) | UNIQUE, NOT NULL | - | 用户名 |
| password | VARCHAR(100) | NOT NULL | - | 密码(BCrypt加密) |
| real_name | VARCHAR(50) | - | NULL | 真实姓名 |
| phone | VARCHAR(20) | UNIQUE | NULL | 手机号 |
| email | VARCHAR(100) | - | NULL | 邮箱 |
| gender | TINYINT | - | 0 | 性别(0-未知, 1-男, 2-女) |
| age | INT | - | NULL | 年龄 |
| id_card | VARCHAR(18) | - | NULL | 身份证号 |
| role | VARCHAR(20) | NOT NULL | 'PATIENT' | 角色(PATIENT/DOCTOR/ADMIN) |
| avatar | VARCHAR(255) | - | NULL | 头像URL |
| status | TINYINT | - | 1 | 状态(0-禁用, 1-正常) |
| last_login_time | DATETIME | - | NULL | 最后登录时间 |
| last_login_ip | VARCHAR(50) | - | NULL | 最后登录IP |
| deleted | TINYINT | - | 0 | 逻辑删除(0-未删除, 1-已删除) |
| create_time | DATETIME | - | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | - | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引设计:**
- PRIMARY KEY: `id`
- UNIQUE KEY: `uk_username` (username)
- UNIQUE KEY: `uk_phone` (phone)
- INDEX: `idx_role` (role)
- INDEX: `idx_status` (status)

## 索引设计

### 索引优化原则

1. **主键索引**: 所有表使用自增BIGINT作为主键
2. **唯一索引**: 用户名、手机号、设备序列号等唯一字段
3. **联合索引**: 根据查询场景创建联合索引
4. **覆盖索引**: 尽量让索引包含查询所需的所有字段

### 关键索引说明

#### sys_user表
```sql
-- 用户登录查询
CREATE INDEX idx_username ON sys_user(username);

-- 角色筛选
CREATE INDEX idx_role ON sys_user(role);

-- 患者列表查询
CREATE INDEX idx_role_status ON sys_user(role, status);
```

#### device表
```sql
-- 用户设备查询
CREATE INDEX idx_user_status ON device(user_id, status);

-- 设备类型筛选
CREATE INDEX idx_type_status ON device(device_type, status);
```

#### health_alert表
```sql
-- 用户预警查询
CREATE INDEX idx_user_status_time ON health_alert(user_id, status, create_time);

-- 未处理预警统计
CREATE INDEX idx_status_level ON health_alert(status, alert_level);
```

## 时序数据设计 (InfluxDB)

### Measurement: heart_rate

**用途**: 存储实时心率监测数据

#### Tags (索引字段)

| Tag | 类型 | 说明 | 示例 |
|-----|------|------|------|
| userId | STRING | 用户ID | "1", "2" |
| deviceId | STRING | 设备ID | "1", "2" |

#### Fields (数据字段)

| Field | 类型 | 说明 | 取值范围 |
|-------|------|------|----------|
| heartRate | FLOAT | 心率值(次/分钟) | 30-250 |
| rrInterval | FLOAT | RR间期(毫秒) | 200-2000 |
| hrv | FLOAT | 心率变异性(毫秒) | 0-200 |
| activityStatus | STRING | 运动状态 | REST/WALK/RUN |
| dataQuality | INTEGER | 数据质量(0-100) | 0-100 |
| isAbnormal | INTEGER | 是否异常 | 0/1 |
| abnormalType | STRING | 异常类型 | TACHYCARDIA/BRADYCARDIA/ARRHYTHMIA |

#### Retention Policy (保留策略)

```sql
-- 原始数据保留30天
CREATE RETENTION POLICY "30days" ON "health-data" DURATION 30d REPLICATION 1 DEFAULT

-- 聚合数据(5分钟平均)保留1年
CREATE RETENTION POLICY "1year" ON "health-data" DURATION 365d REPLICATION 1

-- 统计数据(小时平均)永久保存
CREATE RETENTION POLICY "infinite" ON "health-data" DURATION INF REPLICATION 1
```

#### Continuous Queries (连续查询)

```sql
-- 5分钟平均心率
CREATE CONTINUOUS QUERY "cq_5m_avg" ON "health-data"
BEGIN
  SELECT mean("heartRate") AS "avgHeartRate" 
  INTO "1year"."heart_rate_5m" 
  FROM "heart_rate" 
  GROUP BY time(5m), "userId"
END

-- 小时统计
CREATE CONTINUOUS QUERY "cq_1h_stats" ON "health-data"
BEGIN
  SELECT mean("heartRate") AS "avgHR", 
         max("heartRate") AS "maxHR", 
         min("heartRate") AS "minHR",
         count("heartRate") AS "count"
  INTO "infinite"."heart_rate_1h" 
  FROM "heart_rate" 
  GROUP BY time(1h), "userId"
END
```

### 查询示例

#### 查询最新心率数据
```flux
from(bucket: "health-data")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "heart_rate")
  |> filter(fn: (r) => r.userId == "1")
  |> last()
```

#### 查询平均心率
```flux
from(bucket: "health-data")
  |> range(start: -24h)
  |> filter(fn: (r) => r._measurement == "heart_rate")
  |> filter(fn: (r) => r.userId == "1")
  |> mean(column: "heartRate")
```

#### 检测异常心率
```flux
from(bucket: "health-data")
  |> range(start: -7d)
  |> filter(fn: (r) => r._measurement == "heart_rate")
  |> filter(fn: (r) => r.userId == "1")
  |> filter(fn: (r) => r.isAbnormal == 1)
```

## 数据字典

### 用户角色 (role)

| 值 | 说明 | 权限 |
|----|------|------|
| PATIENT | 患者 | 查看自己的数据、接收预警 |
| DOCTOR | 医生 | 查看患者数据、处理预警、审核报告 |
| ADMIN | 管理员 | 系统管理、用户管理 |

### 设备类型 (device_type)

| 值 | 说明 | 监测参数 |
|----|------|----------|
| ECG | 心电图设备 | 心电信号、RR间期 |
| HR | 心率设备 | 心率、HRV |
| BP | 血压设备 | 收缩压、舒张压 |
| SPO2 | 血氧设备 | 血氧饱和度 |

### 预警类型 (alert_type)

| 值 | 说明 | 触发条件 |
|----|------|----------|
| TACHYCARDIA | 心动过速 | 心率 > 100次/分 |
| BRADYCARDIA | 心动过缓 | 心率 < 60次/分 |
| ARRHYTHMIA | 心律失常 | HRV异常、RR间期不规则 |
| HIGH_BP | 高血压 | 收缩压 > 140 或 舒张压 > 90 |
| LOW_BP | 低血压 | 收缩压 < 90 或 舒张压 < 60 |

### 预警级别 (alert_level)

| 值 | 颜色 | 说明 | 处理方式 |
|----|------|------|----------|
| LOW | 蓝色 | 轻微异常,注意观察 | 自动记录 |
| MEDIUM | 黄色 | 中度异常,建议咨询 | 提醒患者 |
| HIGH | 橙色 | 严重异常,尽快就医 | 通知医生 |
| CRITICAL | 红色 | 危急情况,立即就医 | 紧急通知+拨打急救电话 |

### 报告类型 (report_type)

| 值 | 说明 | 生成频率 | 内容 |
|----|------|----------|------|
| DAILY | 日报 | 每日凌晨生成 | 当日数据统计、趋势分析 |
| WEEKLY | 周报 | 每周一生成 | 周度统计、趋势对比 |
| MONTHLY | 月报 | 每月1号生成 | 月度总结、健康评估 |

## 数据安全

### 1. 敏感数据加密

- **密码**: 使用BCrypt算法加密存储
- **身份证号**: AES-256加密存储
- **Token**: JWT签名验证

### 2. 数据备份策略

```bash
# MySQL全量备份(每日凌晨2点)
0 2 * * * mysqldump -u root -p cardioguard > /backup/mysql_$(date +\%Y\%m\%d).sql

# InfluxDB备份(每周日凌晨3点)
0 3 * * 0 influx backup /backup/influxdb_$(date +\%Y\%m\%d)

# 备份保留30天
find /backup -name "*.sql" -mtime +30 -delete
```

### 3. 数据清理

```sql
-- 清理90天前的日志数据
DELETE FROM health_alert WHERE create_time < DATE_SUB(NOW(), INTERVAL 90 DAY);

-- 清理已删除用户的数据(软删除超过1年)
DELETE FROM sys_user WHERE deleted = 1 AND update_time < DATE_SUB(NOW(), INTERVAL 1 YEAR);
```

## 性能优化

### 1. 查询优化

**避免全表扫描:**
```sql
-- ❌ 错误示例
SELECT * FROM sys_user WHERE YEAR(create_time) = 2024;

-- ✅ 正确示例
SELECT * FROM sys_user WHERE create_time >= '2024-01-01';
```

**使用覆盖索引:**
```sql
-- 查询只包含索引字段
SELECT id, username, role FROM sys_user WHERE role = 'PATIENT';
```

### 2. 分页优化

**大数据量分页:**
```sql
-- 使用游标分页
SELECT * FROM health_alert 
WHERE user_id = 1 AND id < 10000 
ORDER BY id DESC 
LIMIT 20;
```

### 3. 批量操作

**批量插入:**
```sql
INSERT INTO health_alert (user_id, alert_type, alert_level, title) VALUES
(1, 'TACHYCARDIA', 'HIGH', '心动过速'),
(2, 'BRADYCARDIA', 'MEDIUM', '心动过缓'),
(3, 'ARRHYTHMIA', 'LOW', '心律失常');
```

## 监控指标

### 关键指标

1. **数据库连接数**: 监控连接池使用情况
2. **慢查询**: 记录执行时间超过1秒的查询
3. **锁等待**: 监控死锁和长事务
4. **磁盘空间**: 监控数据文件大小
5. **QPS/TPS**: 每秒查询数/事务数

### 告警阈值

| 指标 | 警告 | 严重 |
|------|------|------|
| 连接池使用率 | > 80% | > 95% |
| 慢查询数量 | > 10/min | > 50/min |
| 磁盘使用率 | > 70% | > 90% |
| CPU使用率 | > 70% | > 90% |
| 内存使用率 | > 80% | > 95% |

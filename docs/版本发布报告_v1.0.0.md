## 🎯 发布概述

CardioGuard 360 v1.0.0 是一个**实时心血管健康监护平台**的初始版本,提供了完整的心率监测、异常预警、健康报告等核心功能。

### 发布亮点

✅ **完整的后端系统** - 基于 Spring Boot 3.2.0 + Java 17  
✅ **实时数据推送** - WebSocket 实现毫秒级心率数据推送  
✅ **时序数据存储** - InfluxDB 高效存储和查询海量监测数据  
✅ **多级预警机制** - 智能检测心动过速/过缓等异常情况  
✅ **Docker 一键部署** - 完整的容器化部署方案  
✅ **完善的技术文档** - API文档、部署指南、数据库设计文档

## 🚀 部署方案

### 1. 本地开发环境

**前置要求**:
- JDK 17+
- Maven 3.6+
- MySQL 8.0+
- Redis 7.x
- InfluxDB 2.7+

**部署步骤**:
```bash
# 1. 初始化数据库
mysql -u root -p < database/init.sql

# 2. 配置应用
编辑 application.yml 中的数据库连接信息

# 3. 编译运行
cd cardio-guard-backend
mvn clean package -DskipTests
java -jar target/cardioguard-backend-1.0.0.jar

# 4. 访问API文档
http://localhost:8080/api/swagger-ui.html
```

### 2. Docker一键部署

```bash
# 一键启动所有服务
docker-compose up -d

# 查看运行状态
docker-compose ps

# 查看日志
docker-compose logs -f backend
```

**包含的服务**:
- cardioguard-mysql (MySQL 8.0)
- cardioguard-redis (Redis 7)
- cardioguard-influxdb (InfluxDB 2.7)
- cardioguard-backend (应用服务)

### 3. 生产环境部署

**推荐配置**:
- CPU: 4核心
- 内存: 8GB
- 磁盘: 50GB SSD
- OS: Ubuntu 22.04 LTS

**关键优化**:
- JVM参数: `-Xms2g -Xmx4g`
- 数据库连接池: HikariCP max-pool-size=20
- Nginx反向代理 + HTTPS
- Systemd服务管理

## 🔒 安全特性

### 安全措施

| 层面 | 措施 | 说明 |
|------|------|------|
| 认证 | JWT Token | 无状态认证,24小时过期 |
| 授权 | Spring Security | 基于角色的访问控制 |
| 密码 | BCrypt加密 | 单向哈希,盐值随机 |
| 传输 | HTTPS | 生产环境强制使用 |
| CORS | 跨域配置 | 可配置允许的源 |
| SQL注入 | MyBatis参数化 | 防止SQL注入攻击 |

### 敏感数据处理

- ✅ 密码: BCrypt加密存储
- ⏳ 身份证号: AES-256加密(待实现)
- ✅ Token: JWT签名验证
- ✅ 日志脱敏: 不记录敏感信息

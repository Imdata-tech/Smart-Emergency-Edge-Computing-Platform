## 🎖 技术突破点

### 1. Mock 策略优化

**Axios Mock:**
```typescript
vi.mock('axios', () => ({
  default: {
    post: vi.fn(),
    get: vi.fn()
  }
}))
```

**优势:**
- 隔离外部依赖
- 控制测试场景
- 提高测试速度

### 2. Pinia Store 测试模式

**独立 Store 实例:**
```typescript
beforeEach(() => {
  setActivePinia(createPinia())
  userStore = useUserStore()
})
```

**优势:**
- 每个测试独立
- 避免状态污染
- 易于调试

### 3. 组件挂载测试

**完整插件集成:**
```typescript
wrapper = mount(Login, {
  global: {
    plugins: [router, ElementPlus]
  }
})
```

**优势:**
- 真实环境测试
- 验证插件集成
- 发现兼容问题

### 4. 异步测试处理

**Async/Await 模式:**
```typescript
it('登录成功应该设置 token', async () => {
  await userStore.login('user', 'pass')
  expect(userStore.token).toBe('token')
})
```

**优势:**
- 清晰的异步流程
- 准确的断言时机
- 易于理解

## 📝 最佳实践总结

### 1. 测试命名规范
```typescript
// 格式: it('应该[预期行为]当[条件]', () => {})
it('登录成功应该设置 token 和用户信息', () => {})
it('未登录用户访问受保护路由应该重定向到登录页', () => {})
```

### 2. AAA 模式
```typescript
// Arrange - 准备测试数据
const mockToken = 'test-token'

// Act - 执行被测代码
await userStore.login('user', 'pass')

// Assert - 验证结果
expect(userStore.token).toBe(mockToken)
```

### 3. Mock 外部依赖
```typescript
// Mock API 调用
;(axios.post as any).mockResolvedValueOnce({ data: { data: 'token' } })

// Mock WebSocket
;(global as any).WebSocket = vi.fn(() => mockWebSocket)
```

### 4. 清理副作用
```typescript
beforeEach(() => {
  localStorage.clear()
  setActivePinia(createPinia())
})
```

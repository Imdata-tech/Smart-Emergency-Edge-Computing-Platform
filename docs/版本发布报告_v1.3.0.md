## ✨ 核心功能

### 1. ECG 数据模型

#### EcgData 实体类
**用途**: 存储 ECG 心电图时序数据到 InfluxDB

**核心字段:**
- `waveform` - ECG 波形数据(Base64/JSON)
- `sampleRate` - 采样率(250Hz/500Hz)
- `duration` - 记录时长(秒)
- `prInterval` / `qrsDuration` / `qtInterval` - 心电间期参数
- `axis` - 电轴角度
- `analysisResultId` - 关联分析结果

#### EcgAnalysisResult 实体类
**用途**: 存储 AI 分析结果到 MySQL

**核心字段:**
- `diagnosis` - 诊断结果(NORMAL/ABNORMAL)
- `arrhythmiaType` - 心律失常类型(AFIB/PVC/PAC/ST_ELEVATION)
- `confidence` - AI 置信度(0-1)
- `riskLevel` - 风险等级(LOW/MEDIUM/HIGH/CRITICAL)
- `recommendation` - 建议措施
- `reviewStatus` - 审核状态(PENDING/APPROVED/REJECTED)
- `reviewedBy` / `reviewedAt` / `reviewComment` - 医生审核信息

### 2. AI 分析服务

#### EcgAnalysisService 接口
**核心方法:**
```java
// 单条ECG分析
EcgAnalysisResult analyzeEcg(Long userId, Long deviceId, 
                              String waveform, Integer sampleRate);

// 批量分析
List<EcgAnalysisResult> batchAnalyze(List<Map<String, Object>> ecgDataList);

// 查询历史
List<EcgAnalysisResult> getUserAnalysisHistory(Long userId, 
                                                Integer page, Integer size);

// 医生审核
EcgAnalysisResult reviewResult(Long resultId, Long doctorId, 
                                String reviewStatus, String comment);

// 生成诊断报告
String generateDiagnosisReport(Long resultId);

// 异常统计
Map<String, Object> getAbnormalStatistics(Long userId, 
                                          String startDate, String endDate);
```

#### AI 模拟算法
**实现逻辑:**
- 20% 概率检测到异常
- 支持 4 种心律失常类型:
  - **AFIB** (房颤) - 高风险
  - **PVC** (室性早搏) - 中风险
  - **PAC** (房性早搏) - 低风险
  - **ST_ELEVATION** (ST段抬高) - 危急风险
- AI 置信度: 85%-99%
- 自动生成个性化建议措施

### 3. REST API 接口

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 分析ECG | POST | `/api/ecg/analyze` | 单条ECG数据分析 |
| 批量分析 | POST | `/api/ecg/batch-analyze` | 批量ECG数据分析 |
| 查询历史 | GET | `/api/ecg/history` | 获取用户分析历史 |
| 结果详情 | GET | `/api/ecg/result/{id}` | 获取分析结果详情 |
| 医生审核 | PUT | `/api/ecg/review/{id}` | 医生审核分析结果 |
| 诊断报告 | GET | `/api/ecg/report/{id}` | 生成Markdown诊断报告 |
| 异常统计 | GET | `/api/ecg/statistics/abnormal` | 统计用户异常情况 |

### 4. 医生审核流程

**状态机:**
```
PENDING (待审核)
    ↓
APPROVED (已通过) 或 REJECTED (已拒绝)
```

**审核字段:**
- `needsReview` - 是否需要审核(异常结果自动标记为1)
- `reviewStatus` - 审核状态
- `reviewedBy` - 审核医生ID
- `reviewedAt` - 审核时间
- `reviewComment` - 审核意见

## 📊 技术实现

### 数据库设计

**ecg_analysis_result 表结构:**
```sql
CREATE TABLE `ecg_analysis_result` (
  `id` BIGINT AUTO_INCREMENT PRIMARY KEY,
  `user_id` BIGINT NOT NULL,
  `device_id` BIGINT NOT NULL,
  `ecg_data_id` VARCHAR(100) NOT NULL,
  `diagnosis` VARCHAR(20) NOT NULL,
  `arrhythmia_type` VARCHAR(50),
  `confidence` DECIMAL(5,4) NOT NULL,
  `model_version` VARCHAR(50) NOT NULL,
  `analysis_report` TEXT,
  `risk_level` VARCHAR(20) NOT NULL,
  `recommendation` TEXT,
  `needs_review` TINYINT DEFAULT 0,
  `review_status` VARCHAR(20) DEFAULT 'PENDING',
  `reviewed_by` BIGINT,
  `reviewed_at` DATETIME,
  `review_comment` TEXT,
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  KEY `idx_user_id` (`user_id`),
  KEY `idx_diagnosis` (`diagnosis`),
  KEY `idx_risk_level` (`risk_level`),
  KEY `idx_review_status` (`review_status`)
);
```

**索引优化:**
- `idx_user_id` - 快速查询用户历史
- `idx_diagnosis` - 筛选正常/异常结果
- `idx_risk_level` - 按风险等级统计
- `idx_review_status` - 查询待审核结果

## 🚀 部署指南

### 1. 数据库初始化

```bash
# 执行SQL脚本
mysql -u root -p cardioguard < database/ecg_analysis_init.sql

# 验证表创建
mysql -u root -p cardioguard -e "SHOW TABLES LIKE 'ecg_analysis_result';"
```

### 2. 后端启动

```bash
cd cardio-guard-backend
mvn clean package
java -jar target/cardioguard-backend-1.0.0.jar
```


## ⚠️ 已知限制

### 当前版本限制

1. **AI 模型为模拟实现**
   - 目前使用随机数模拟 AI 分析
   - 未集成真实的深度学习模型
   - 准确率依赖于模拟算法

2. **ECG 波形未可视化**
   - 前端尚未实现 ECG 波形显示
   - 需要集成 ECharts 或其他图表库

3. **实时监测未实现**
   - 缺少 WebSocket 实时 ECG 推送
   - 需要扩展 HealthMonitorWebSocketHandler

### 后续改进方向

1. **集成真实 AI 模型**
   - TensorFlow/PyTorch 模型集成
   - 使用公开数据集训练(ECG-ID Database)
   - 提升准确率和召回率

2. **前端可视化**
   - ECG 波形实时显示
   - 交互式标注功能
   - 多导联同步显示

3. **性能优化**
   - 批量分析异步处理
   - 分析结果缓存(Redis)
   - 数据库查询优化
